# 组合导航卡尔曼滤波详解

## 1. 系统概述

组合导航系统（LooselyCoupled）采用松耦合架构，将GNSS位置/速度观测与INS递推结果进行融合，实现高精度、高可靠性的导航定位。

### 1.1 系统特点

- **松耦合架构**：GNSS和INS独立解算，通过卡尔曼滤波融合
- **延时更新**：支持GNSS数据延时到达的处理
- **多种约束**：支持ZUPT、NHC、里程计等辅助观测

## 2. 惯导机械编排 (INS Mechanization)

### 2.1 完整的机械编排方程

组合导航采用高精度的惯导机械编排，考虑地球自转、有害加速度等因素。

#### 2.1.1 姿态更新

采用四元数更新：

$$\boldsymbol{q}_n^b(k) = \boldsymbol{q}_{n(k-1)}^{n(k)} \otimes \boldsymbol{q}_{n}^b(k-1) \otimes \boldsymbol{q}_{b(k-1)}^{b(k)}$$

**载体系旋转四元数**：
$$\boldsymbol{q}_{b(k-1)}^{b(k)} \approx \begin{bmatrix} 1 \\ \frac{1}{2}\boldsymbol{\phi}_k \end{bmatrix}$$

其中考虑圆锥效应：
$$\boldsymbol{\phi}_k = \Delta\boldsymbol{\theta}_k + \frac{1}{12}\Delta\boldsymbol{\theta}_{k-1} \times \Delta\boldsymbol{\theta}_k$$

**导航系旋转四元数**：
$$\boldsymbol{q}_{n(k-1)}^{n(k)} = \text{rotvec2quat}(-(\boldsymbol{\omega}_{ie}^n + \boldsymbol{\omega}_{en}^n) \cdot \Delta t)$$

其中：
- $\boldsymbol{\omega}_{ie}^n = [\omega_e \cos\varphi, 0, -\omega_e \sin\varphi]^T$：地球自转角速度在n系投影
- $\boldsymbol{\omega}_{en}^n = [v_E/(R_N+h), -v_N/(R_M+h), -v_E\tan\varphi/(R_N+h)]^T$：由位移引起的n系转动

#### 2.1.2 速度更新

$$\boldsymbol{v}_n^{k} = \boldsymbol{v}_n^{k-1} + \Delta\boldsymbol{v}_{sf} + \Delta\boldsymbol{v}_{cor/g}$$

**比力速度增量**：
$$\Delta\boldsymbol{v}_{sf} = \boldsymbol{C}_{nn} \cdot \boldsymbol{C}_b^{n(k-1)} \cdot \Delta\boldsymbol{v}_{fb}$$

其中：
$$\boldsymbol{C}_{nn} = \boldsymbol{I} - [\frac{1}{2}(\boldsymbol{\omega}_{ie}^n + \boldsymbol{\omega}_{en}^n)\Delta t]_\times$$

**比力积分考虑划桨效应**：
$$\Delta\boldsymbol{v}_{fb} = \Delta\boldsymbol{v} + \frac{1}{2}\Delta\boldsymbol{\theta} \times \Delta\boldsymbol{v} + \frac{1}{12}(\Delta\boldsymbol{\theta}^{k-1} \times \Delta\boldsymbol{v}^k + \Delta\boldsymbol{v}^{k-1} \times \Delta\boldsymbol{\theta}^k)$$

**哥氏力和重力增量**：
$$\Delta\boldsymbol{v}_{cor/g} = (\boldsymbol{g}_n - (2\boldsymbol{\omega}_{ie}^n + \boldsymbol{\omega}_{en}^n) \times \boldsymbol{v}_n^{mid}) \cdot \Delta t$$

#### 2.1.3 位置更新

采用四元数表示的位置更新：

$$\boldsymbol{q}_{ne}(k) = \boldsymbol{q}_e \otimes \boldsymbol{q}_{ne}(k-1) \otimes \boldsymbol{q}_n$$

其中：
- $\boldsymbol{q}_n = \text{rotvec2quat}((\boldsymbol{\omega}_{ie}^n + \boldsymbol{\omega}_{en}^n)\Delta t)$
- $\boldsymbol{q}_e = \text{rotvec2quat}([0, 0, -\omega_e\Delta t]^T)$

高度更新：
$$h_k = h_{k-1} - v_D^{mid} \cdot \Delta t$$

**代码实现** (`Ins.c - insMech()`):
```c
void insMech(LcData *lcdata) {
    double dt = lcdata->imudata.dt;
    
    /* 速度更新 */
    // 旋转效应和划桨效应
    crossProduct(lcdata->imudata.dthetacur, lcdata->imudata.dvelcur, temp1);
    crossProduct(lcdata->imudata.dthetapre, lcdata->imudata.dvelcur, temp2);
    crossProduct(lcdata->imudata.dvelpre, lcdata->imudata.dthetacur, temp3);
    
    for (i = 0; i < 3; i++) {
        dvfb[i] = lcdata->imudata.dvelcur[i] + 0.5*temp1[i] + (temp2[i]+temp3[i])/12.0;
        midvel[i] = lcdata->pvacur.vel[i] + 0.5*lcdata->dvn[i];
    }
    
    // 地球自转和牵连角速度
    temp1[0] = WGS84_WIE * cos(lcdata->pvacur.pos[0]);
    temp1[1] = 0;
    temp1[2] = -WGS84_WIE * sin(lcdata->pvacur.pos[0]);
    
    radiusmn(lcdata->pvacur.pos[0], &lcdata->rm, &lcdata->rn);
    temp2[0] = lcdata->pvacur.vel[1] / (lcdata->rn + lcdata->pvacur.pos[2]);
    temp2[1] = -lcdata->pvacur.vel[0] / (lcdata->rm + lcdata->pvacur.pos[2]);
    temp2[2] = -temp2[0] * tan(lcdata->pvacur.pos[0]);
    
    // 比力转换到导航系
    matrixMul(3, 3, cn, 3, 3, lcdata->cbn, cc);
    matrixMul(3, 3, cc, 3, 1, dvfb, lcdata->fvn);
    
    // 重力和哥氏力
    crossProduct(temp2, midvel, temp3);  // (2wie + wen) × v
    temp2[2] = getGravity(midlat, midheight);
    
    for (i = 0; i < 3; i++) {
        lcdata->dvn[i] = lcdata->fvn[i] + (temp2[i] - temp3[i]) * dt;
        lcdata->pvacur.vel[i] += lcdata->dvn[i];
    }
    
    /* 位置更新 - 使用四元数 */
    // ... 四元数更新代码
    
    /* 姿态更新 */
    // 圆锥效应
    crossProduct(lcdata->imudata.dthetapre, lcdata->imudata.dthetacur, temp1);
    for (i = 0; i < 3; i++) {
        temp2[i] = lcdata->imudata.dthetacur[i] + temp1[i]/12.0;
    }
    rotvec2quat(temp2, q2);  // q_b
    
    // n系旋转
    temp3[0] = -(lcdata->iewn[0] + lcdata->enwn[0]) * dt;
    temp3[1] = -(lcdata->iewn[1] + lcdata->enwn[1]) * dt;
    temp3[2] = -(lcdata->iewn[2] + lcdata->enwn[2]) * dt;
    rotvec2quat(temp3, q1);
    
    quatmul(lcdata->qbn, q2, q3);
    quatmul(q1, q3, lcdata->qbn);
    quatnorm(lcdata->qbn);
    quat2dcm(lcdata->qbn, lcdata->cbn);
    dcm2euler(lcdata->cbn, lcdata->pvacur.att);
}
```

## 3. 卡尔曼滤波器设计

### 3.1 状态向量

组合导航采用15维或16维状态向量：

$$\boldsymbol{x} = \begin{bmatrix} \delta\boldsymbol{r} \\ \delta\boldsymbol{v} \\ \boldsymbol{\phi} \\ \boldsymbol{b}_g \\ \boldsymbol{b}_a \\ \delta k_{odo} \end{bmatrix}$$

| 索引 | 状态量 | 说明 | 单位 |
|------|--------|------|------|
| 0-2 | $\delta\boldsymbol{r}$ | 位置误差 | rad, rad, m |
| 3-5 | $\delta\boldsymbol{v}$ | 速度误差 | m/s |
| 6-8 | $\boldsymbol{\phi}$ | 姿态误差 | rad |
| 9-11 | $\boldsymbol{b}_g$ | 陀螺仪零偏 | rad/s |
| 12-14 | $\boldsymbol{b}_a$ | 加速度计零偏 | m/s² |
| 15 | $\delta k_{odo}$ | 里程计比例因子误差 | - |

### 3.2 状态转移矩阵 Φ

连续时间状态方程：
$$\dot{\boldsymbol{x}} = \boldsymbol{F}\boldsymbol{x} + \boldsymbol{G}\boldsymbol{w}$$

离散化后的状态转移矩阵：
$$\boldsymbol{\Phi} \approx \boldsymbol{I} + \boldsymbol{F} \cdot \Delta t$$

#### 3.2.1 F矩阵结构

$$\boldsymbol{F} = \begin{bmatrix} \boldsymbol{F}_{rr} & \boldsymbol{F}_{rv} & \boldsymbol{0} & \boldsymbol{0} & \boldsymbol{0} & \boldsymbol{0} \\ \boldsymbol{F}_{vr} & \boldsymbol{F}_{vv} & \boldsymbol{F}_{v\phi} & \boldsymbol{0} & \boldsymbol{C}_b^n & \boldsymbol{0} \\ \boldsymbol{0} & \boldsymbol{F}_{\phi v} & \boldsymbol{F}_{\phi\phi} & -\boldsymbol{C}_b^n & \boldsymbol{0} & \boldsymbol{0} \\ \boldsymbol{0} & \boldsymbol{0} & \boldsymbol{0} & \boldsymbol{F}_{bg} & \boldsymbol{0} & \boldsymbol{0} \\ \boldsymbol{0} & \boldsymbol{0} & \boldsymbol{0} & \boldsymbol{0} & \boldsymbol{F}_{ba} & \boldsymbol{0} \\ \boldsymbol{0} & \boldsymbol{0} & \boldsymbol{0} & \boldsymbol{0} & \boldsymbol{0} & 0 \end{bmatrix}$$

#### 3.2.2 位置误差相关项

$$\boldsymbol{F}_{rr} = -[\boldsymbol{\omega}_{en}^n]_\times$$

$$\boldsymbol{F}_{rv} = \boldsymbol{I}_{3\times3}$$

#### 3.2.3 速度误差相关项

$$\boldsymbol{F}_{vr} = \begin{bmatrix} -g/(R_M+h) & 0 & 0 \\ 0 & -g/(R_N+h) & 0 \\ 0 & 0 & 2g/(\sqrt{R_MR_N}+h) \end{bmatrix}$$

$$\boldsymbol{F}_{vv} = -[(2\boldsymbol{\omega}_{ie}^n + \boldsymbol{\omega}_{en}^n)]_\times$$

$$\boldsymbol{F}_{v\phi} = [\boldsymbol{f}^n]_\times$$

#### 3.2.4 姿态误差相关项

$$\boldsymbol{F}_{\phi\phi} = -[(\boldsymbol{\omega}_{ie}^n + \boldsymbol{\omega}_{en}^n)]_\times$$

$$\boldsymbol{F}_{\phi bg} = -\boldsymbol{C}_b^n$$

#### 3.2.5 IMU零偏相关项

采用一阶Gauss-Markov过程：

$$\boldsymbol{F}_{bg} = -\frac{1}{\tau_g}\boldsymbol{I}_{3\times3}$$

$$\boldsymbol{F}_{ba} = -\frac{1}{\tau_a}\boldsymbol{I}_{3\times3}$$

**代码实现** (`Lc.c - lcUpdatePhi()`):
```c
static void lcUpdatePhi(const LcConfig *lcconfig, LcData *lcdata, double *phi) {
    double dt = lcdata->imudata.dt;
    double gdt = lcdata->gravity * dt;
    double rmh = lcdata->rm + lcdata->pvacur.pos[2];
    double rnh = lcdata->rn + lcdata->pvacur.pos[2];
    
    /* 位置误差 */
    for (i = 0; i < 3; i++) {
        tmp[i] = -lcdata->enwn[i] * dt;  // -(wen×) * dt
    }
    vecTransSymm(tmp, tmp + 6);
    for (i = 0; i < 3; i++) {
        for (j = 0; j < 3; j++) {
            phi[i*rank + j] = (tmp + 6)[i*3 + j];
        }
        phi[i*rank + i] = 1;
        phi[i*rank + i + 3] = dt;  // F_rv = I
    }
    
    /* 速度误差 */
    (tmp + 6)[0] = -gdt / rmh;
    (tmp + 6)[4] = -gdt / rnh;
    (tmp + 6)[8] = 2 * gdt / (sqrt(lcdata->rm * lcdata->rn) + lcdata->pvacur.pos[2]);
    
    for (i = 0; i < 3; i++) {
        tmp[i] = -(2*lcdata->iewn[i] + lcdata->enwn[i]) * dt;  // -(2wie+wen)× * dt
        (tmp + 3)[i] = lcdata->fvn[i] * dt;  // fn× * dt
    }
    vecTransSymm(tmp, tmp + 15);      // F_vv
    vecTransSymm(tmp + 3, tmp + 24);  // F_vphi
    
    for (i = 0; i < 3; i++) {
        for (j = 0; j < 3; j++) {
            phi[(i+3)*rank + j] = (tmp + 6)[i*3 + j];       // F_vr
            phi[(i+3)*rank + j + 3] = (tmp + 15)[i*3 + j];  // F_vv
            phi[(i+3)*rank + j + 6] = (tmp + 24)[i*3 + j];  // F_vphi
            phi[(i+3)*rank + j + 12] = lcdata->cbn[i*3 + j] * dt;  // Cbn
        }
        phi[(i+3)*rank + i + 3] = 1;
    }
    
    /* 姿态误差 */
    for (i = 0; i < 3; i++) {
        tmp[i] = -(lcdata->iewn[i] + lcdata->enwn[i]) * dt;  // -(wie+wen)× * dt
    }
    vecTransSymm(tmp, tmp + 3);
    for (i = 0; i < 3; i++) {
        for (j = 0; j < 3; j++) {
            phi[(i+6)*rank + j + 6] = (tmp + 3)[i*3 + j];  // F_phiphi
            phi[(i+6)*rank + j + 9] = -lcdata->cbn[i*3 + j] * dt;  // -Cbn
        }
        phi[(i+6)*rank + i + 6] = 1;
    }
    
    /* IMU零偏 - 一阶Markov */
    for (i = 9; i < 12; i++) {
        phi[rank*i + i] = 1 - dt / lcconfig->imumodel.gyrbiascortime;
        phi[rank*(i+3) + i + 3] = 1 - dt / lcconfig->imumodel.accbiascortime;
    }
    
    /* 里程计比例因子 */
    if (lcconfig->isuseodo) {
        phi[KALMANXI * rank + KALMANXI] = 1;
    }
}
```

### 3.3 过程噪声矩阵 Q

连续时间过程噪声谱密度矩阵：

$$\boldsymbol{Q}_c = \text{diag}([0, 0, 0, \sigma_{a}^2, \sigma_{a}^2, \sigma_{a}^2, \sigma_g^2, \sigma_g^2, \sigma_g^2, \frac{2\sigma_{bg}^2}{\tau_g}, \frac{2\sigma_{bg}^2}{\tau_g}, \frac{2\sigma_{bg}^2}{\tau_g}, \frac{2\sigma_{ba}^2}{\tau_a}, \frac{2\sigma_{ba}^2}{\tau_a}, \frac{2\sigma_{ba}^2}{\tau_a}, \sigma_{odo}^2])$$

其中：
- $\sigma_a$：加速度计速度随机游走 (VRW)
- $\sigma_g$：陀螺仪角度随机游走 (ARW)
- $\sigma_{bg}$：陀螺仪零偏标准差
- $\sigma_{ba}$：加速度计零偏标准差
- $\tau_g, \tau_a$：相关时间

**离散化**：
$$\boldsymbol{Q}_k \approx \frac{\Delta t}{2}(\boldsymbol{\Phi}\boldsymbol{Q}_c + \boldsymbol{Q}_c\boldsymbol{\Phi}^T)$$

**代码实现** (`Lc.c - lcInitial()`):
```c
for (int k = 0; k < 3; k++) {
    /* 设置Qm阵 */
    lcdata->qmat[0 + k] = 0;  // 位置
    lcdata->qmat[3 + k] = lcconfig->imumodel.accvrw * lcconfig->imumodel.accvrw;  // 速度
    lcdata->qmat[6 + k] = lcconfig->imumodel.gyrarw * lcconfig->imumodel.gyrarw;  // 姿态
    lcdata->qmat[9 + k] = 2 * lcconfig->imumodel.gyrbiasstd * lcconfig->imumodel.gyrbiasstd 
                          / lcconfig->imumodel.gyrbiascortime;  // 陀螺零偏
    lcdata->qmat[12 + k] = 2 * lcconfig->imumodel.accbiasstd * lcconfig->imumodel.accbiasstd 
                           / lcconfig->imumodel.accbiascortime;  // 加速度计零偏
}
```

### 3.4 GNSS观测更新

#### 3.4.1 观测模型

GNSS提供位置观测：

$$\boldsymbol{z}_{GNSS} = \boldsymbol{r}_{INS} - \boldsymbol{r}_{GNSS}$$

考虑天线杆臂：

$$\boldsymbol{r}_{INS} = \boldsymbol{r}_{IMU} + \boldsymbol{C}_b^n \cdot \boldsymbol{l}_{ant}$$

其中 $\boldsymbol{l}_{ant}$ 为天线杆臂在载体系下的坐标。

#### 3.4.2 观测矩阵 H

$$\boldsymbol{H}_{GNSS} = \begin{bmatrix} \boldsymbol{I}_3 & \boldsymbol{0}_3 & [\boldsymbol{C}_b^n \cdot \boldsymbol{l}_{ant}]_\times & \boldsymbol{0}_3 & \boldsymbol{0}_3 & 0 \end{bmatrix}_{3\times16}$$

对于第三维（高度），需要注意符号：
$$H_{2,2} = -1$$

#### 3.4.3 观测噪声矩阵 R

$$\boldsymbol{R}_{GNSS} = \text{diag}([\sigma_N^2, \sigma_E^2, \sigma_D^2])$$

标准差来自GNSS接收机输出的定位精度。

**代码实现** (`Lc.c - lcGnssUpdate()`):
```c
static bool lcGnssUpdate(const LcConfig *lcconfig, LcData *lcdata) {
    double zgnss[3], leveranti[9];
    
    // 计算天线杆臂在n系下的投影
    matrixMul(3, 3, lcdata->cbnud, 3, 1, lcconfig->antlever, tmp3);
    vecTransSymm(tmp3, leveranti);  // 杆臂反对称矩阵
    
    // 杆臂补偿后的位置
    dm2drad(rm, rn, lcdata->pvaud.pos, tmp3, tmp3);
    matrixAdd(3, 1, lcdata->pvaud.pos, tmp3, tmp3);
    
    // 计算新息 (单位：米)
    zgnss[0] = tmp3[0] - lcdata->gnssdata.pos[0];
    zgnss[1] = tmp3[1] - lcdata->gnssdata.pos[1];
    zgnss[2] = lcdata->gnssdata.pos[2] - tmp3[2];  // 高度符号相反
    drad2dm(rm, rn, lcdata->pvaud.pos, zgnss, zgnss);
    
    // 构建观测矩阵
    for (i = 0; i < 3; i++) {
        lcdata->zmat[lcdata->updaterows] = zgnss[i];
        if (i < 2) {
            lcdata->hmat[lcdata->updaterows * rank + i] = 1;
        } else {
            lcdata->hmat[lcdata->updaterows * rank + i] = -1;  // 高度
        }
        // 杆臂项
        for (j = 0; j < 3; j++) {
            lcdata->hmat[lcdata->updaterows * rank + j + 6] = leveranti[i*3 + j];
        }
        lcdata->rmat[lcdata->updaterows] = lcdata->gnssdata.posstd[i];
        lcdata->updaterows += 1;
    }
    
    // 延时更新
    lcDelayUpdateFeedback(lcdata);
}
```

### 3.5 卡尔曼滤波更新方程

**预测步骤**：
$$\begin{align}
\hat{\boldsymbol{x}}_k^- &= \boldsymbol{\Phi}_k \hat{\boldsymbol{x}}_{k-1} \\[8pt]
\boldsymbol{P}_k^- &= \boldsymbol{\Phi}_k \boldsymbol{P}_{k-1} \boldsymbol{\Phi}_k^T + \boldsymbol{Q}_k
\end{align}$$

**更新步骤**：
$$\begin{align}
\boldsymbol{K}_k &= \boldsymbol{P}_k^- \boldsymbol{H}_k^T (\boldsymbol{H}_k \boldsymbol{P}_k^- \boldsymbol{H}_k^T + \boldsymbol{R}_k)^{-1} \\[8pt]
\hat{\boldsymbol{x}}_k &= \hat{\boldsymbol{x}}_k^- + \boldsymbol{K}_k (\boldsymbol{z}_k - \boldsymbol{H}_k \hat{\boldsymbol{x}}_k^-) \\[8pt]
\boldsymbol{P}_k &= (\boldsymbol{I} - \boldsymbol{K}_k \boldsymbol{H}_k) \boldsymbol{P}_k^- (\boldsymbol{I} - \boldsymbol{K}_k \boldsymbol{H}_k)^T + \boldsymbol{K}_k \boldsymbol{R}_k \boldsymbol{K}_k^T
\end{align}$$

注：协方差更新采用Joseph形式，保证数值稳定性。

**代码实现** (`Lc.c - lcKalmanUpdate()`):
```c
static void lcKalmanUpdate(LcData *lcdata, int rows) {
    /* K = P * H' * inv(H * P * H' + R) */
    matrixTranspose(rows, rank, lcdata->hmat, tmp0X2);  // H'
    matrixMul(rank, rank, lcdata->pmatud, rank, rows, tmp0X2, tmp0X2 + rowsn);  // P*H'
    matrixMul(rows, rank, lcdata->hmat, rank, rows, tmp0X2 + rowsn, tmp1X2);  // H*P*H'
    
    for (i = 0; i < rows; i++) {
        tmp1X2[i + rows*i] += lcdata->rmat[i];  // H*P*H' + R
    }
    
    matrixCholesky(tmp1X2, rows);  // Cholesky分解
    matrixInverse(rows, tmp1X2, ...);  // 求逆
    matrixMul(rank, rows, tmp0X2 + rowsn, rows, rows, tmp2X2, tmp1X2);  // K
    
    /* X = X + K * (Z - H*X) */
    matrixMul(rows, rank, lcdata->hmat, rank, 1, lcdata->xmatud, tmp0X2);  // H*X
    for (i = 0; i < rows; i++) {
        lcdata->zmat[i] -= tmp0X2[i];  // Z - H*X
    }
    matrixMul(rank, rows, tmp1X2, rows, 1, lcdata->zmat, tmp0X2);  // K*(Z-H*X)
    matrixAdd(rank, 1, lcdata->xmatud, tmp0X2, lcdata->xmatud);
    
    /* P = (I - K*H) * P * (I - K*H)' + K*R*K' */
    matrixMul(rank, rows, tmp1X2, rows, rank, lcdata->hmat, tmp2X2);  // K*H
    for (i = 0; i < rank; i++) {
        for (j = 0; j < rank; j++) {
            tmp2X2[i*rank + j] = -tmp2X2[i*rank + j];
        }
        tmp2X2[i*rank + i] += 1;  // I - K*H
    }
    
    matrixMul(rank, rank, tmp2X2, rank, rank, lcdata->pmatud, tmp0X2);
    matrixTranspose(rank, rank, tmp2X2, lcdata->pmatud);
    matrixMul(rank, rank, tmp0X2, rank, rank, lcdata->pmatud, tmp2X2);
    
    matrixMulSymm(rank, rows, tmp1X2, lcdata->rmat, tmp0X2);  // K*R*K'
    matrixAdd(rank, rank, tmp2X2, tmp0X2, lcdata->pmatud);
}
```

### 3.6 状态反馈

将估计的误差状态反馈到导航结果：

**位置反馈**：采用旋转矢量方式
$$\boldsymbol{q}_{ne}^{new} = \boldsymbol{q}_{ne} \otimes \boldsymbol{q}(\delta\boldsymbol{r})$$

**速度反馈**：
$$\boldsymbol{v}^{new} = \boldsymbol{v} - \delta\boldsymbol{v}$$

**姿态反馈**：
$$\boldsymbol{q}_{bn}^{new} = \boldsymbol{q}(\boldsymbol{\phi}) \otimes \boldsymbol{q}_{bn}$$

**IMU零偏反馈**：
$$\boldsymbol{b}^{new} = \boldsymbol{b} + \delta\boldsymbol{b}$$

**代码实现** (`Lc.c - lcKalmanFeedback()`):
```c
static void lcKalmanFeedback(LcData *lcdata) {
    /* 位置反馈 */
    dpos2rvec(lcdata->rm, lcdata->rn, lcdata->pvacur.pos, lcdata->xmatpd, rv);
    for (k = 0; k < 3; k++) rv1[k] = -rv[k];
    rotvec2quat(rv1, dq);
    quatmul(lcdata->qne, dq, rv1);
    memcpy(lcdata->qne, rv1, sizeof(double)*4);
    quat2pos(lcdata->qne, &lcdata->pvacur.pos[0], &lcdata->pvacur.pos[1]);
    lcdata->pvacur.pos[2] += lcdata->xmatpd[2];
    
    /* 速度反馈 */
    vecTransSymm(rv, cprv);
    matrixAdd(3, 3, eye, cprv, ccn);
    dv[k] = lcdata->pvacur.vel[k] - lcdata->xmatpd[k + 3];
    matrixMul(3, 3, ccn, 3, 1, dv, lcdata->pvacur.vel);
    
    /* 姿态反馈 */
    dtheta[0] = lcdata->xmatpd[6] + rv[0];
    dtheta[1] = lcdata->xmatpd[7] + rv[1];
    dtheta[2] = lcdata->xmatpd[8] + rv[2];
    rotvec2quat(dtheta, dq);
    quatmul(dq, lcdata->qbn, rv1);
    memcpy(lcdata->qbn, rv1, sizeof(double)*4);
    quat2dcm(lcdata->qbn, lcdata->cbn);
    dcm2euler(lcdata->cbn, lcdata->pvacur.att);
    
    /* IMU零偏反馈 */
    for (k = 0; k < 3; k++) {
        lcdata->imudata.gyrbias[k] += lcdata->xmatpd[9 + k];
        lcdata->imudata.accbias[k] += lcdata->xmatpd[12 + k];
    }
    
    /* 里程计比例因子反馈 */
    lcdata->imudata.odoscale += lcdata->xmatpd[KALMANXI];
}
```

## 4. 新息抗差

### 4.1 原理

当新息过大时（可能存在粗差），降低观测权重或剔除观测：

$$|\boldsymbol{z}|^2 > k^2 \cdot (\boldsymbol{P}^- + \boldsymbol{R})$$

其中 $k$ 为阈值因子（代码中使用6倍）。

**代码实现** (`Lc.c`):
```c
if (lcconfig->isusegnssrobust) {
    // 预测位置标准差小于3倍观测位置标准差
    // 且新息大于6倍新息标准差，则认为是粗差
    if (((lcdata->pmatud[0] < 9 * lcdata->gnssdata.posstd[0]) ||
         (...)) &&
        ((zgnss[0] * zgnss[0] > 36 * (lcdata->pmatud[0] + lcdata->gnssdata.posstd[0])) ||
         (...))) {
        lcdata->gnssdata.mode = 3;  // 标记为无效
    }
}
```

## 5. 延时更新机制

### 5.1 原理

GNSS数据通常有延时，需要将当前时刻的预测状态和协方差反推到GNSS观测时刻，更新后再传播回当前时刻。

### 5.2 实现

累积从观测时刻到当前时刻的状态转移矩阵 $\boldsymbol{\Phi}$ 和过程噪声 $\boldsymbol{Q}$：

```c
if (lcdata->iswaitupdate) {
    /* 累计Q */
    lcUpdatePhiPm(lcconfig, lcdata, tmp0X2, lcdata->qkmat, tmp1X2);
    lcUpdatePhiPmPhit(lcconfig, lcdata, tmp1X2, tmp0X2, lcdata->qkmat);
    matrixAdd(rank, rank, lcdata->qkmat, tmp2X2, lcdata->qkmat);
    
    /* 累计PHI */
    lcUpdatePhiPm(lcconfig, lcdata, tmp0X2, lcdata->phi, tmp1X2);
    memcpy(lcdata->phi, tmp1X2, sizeof(double) * rank * rank);
}
```

延时更新时，将更新后的状态和协方差传播回当前时刻：
```c
void lcDelayUpdateFeedback(LcData *lcdata) {
    /* 卡尔曼更新 */
    lcKalmanUpdate(lcdata, lcdata->updaterows);
    
    /* 状态转移到当前历元 */
    matrixMul(rank, rank, lcdata->phi, rank, 1, lcdata->xmatud, lcdata->xmatpd);
    
    /* 协方差转移 */
    matrixMul(rank, rank, lcdata->phi, rank, rank, lcdata->pmatud, tmp0X2);
    matrixTranspose(rank, rank, lcdata->phi, tmp1X2);
    matrixMul(rank, rank, tmp0X2, rank, rank, tmp1X2, lcdata->pmatpd);
    matrixAdd(rank, rank, lcdata->pmatpd, lcdata->qkmat, lcdata->pmatpd);
    
    /* 反馈更新 */
    lcKalmanFeedback(lcdata);
}
```

## 6. 组合导航与纯DR滤波器对比

### 6.1 两套滤波器的区别

本项目中存在两套滤波器：

| 特性 | 组合导航 (Lc.c) | 纯DR (Kalman.cpp) |
|------|----------------|-------------------|
| 滤波器类型 | 扩展卡尔曼滤波 (EKF) | 线性卡尔曼滤波 (KF) |
| 状态维度 | 16维或17维 | 15维 |
| 状态类型 | 误差状态 | 误差状态 |
| 观测量 | GNSS位置 + 速度约束 | 仅车速约束 |
| 状态转移矩阵 | 每步根据姿态重新计算 | 简化的稀疏矩阵 |
| 非线性处理 | 完整处理地球曲率等 | 假设短时线性 |

### 6.2 为什么都是扩展卡尔曼但状态是误差？

实际上两者都是**误差状态卡尔曼滤波（Error-State Kalman Filter, ESKF）**：

- **组合导航**：完整的ESKF，考虑地球曲率、科氏力等非线性因素
- **纯DR**：简化的ESKF，假设短时间内系统近似线性

### 6.3 为什么使用误差状态而非直接估计状态？

使用误差状态而非直接状态估计有以下优势：

#### 6.3.1 避免奇异性问题

直接使用姿态（如欧拉角）作为状态会遇到万向锁问题。误差状态使用小角度近似：
$$\boldsymbol{\phi} \approx \begin{bmatrix} \phi_{roll} \\ \phi_{pitch} \\ \phi_{yaw} \end{bmatrix}$$

小角度条件下三轴解耦，不存在奇异性。

#### 6.3.2 误差状态接近零，线性化更准确

状态方程在工作点附近线性化：
$$\dot{\boldsymbol{x}} = \boldsymbol{F}(\boldsymbol{x}_0) \cdot \delta\boldsymbol{x} + ...$$

误差状态 $\delta\boldsymbol{x}$ 始终在零附近，高阶项可忽略，线性化精度高。

#### 6.3.3 计算效率高

误差状态每次更新后被清零（反馈到名义状态），协方差矩阵数值稳定，减少数值误差累积。

#### 6.3.4 与惯导机械编排天然配合

惯导机械编排输出名义状态（位置、速度、姿态），滤波器只需估计其误差：

```
名义状态 = INS机械编排输出
真实状态 = 名义状态 - 误差状态
```

## 7. 误差状态卡尔曼滤波原理详解

### 7.1 误差是从哪里来的？

#### 7.1.1 误差状态的初始化

误差状态初始化为零向量，但协方差矩阵不为零：

```c
// 误差状态初始化
memset(lcdata->xmatpd, 0, sizeof(lcdata->xmatpd));  // x = 0

// 协方差矩阵初始化（表示不确定性）
lcdata->pmatpd[0] = lcconfig->initpvastd[0] * lcconfig->initpvastd[0];  // 位置不确定性
lcdata->pmatpd[17] = lcconfig->initpvastd[1] * lcconfig->initpvastd[1];
// ...
```

**解释**：初始时假设INS状态是准确的（误差为0），但对这个"准确"存在不确定性（P≠0）。

#### 7.1.2 预测阶段：误差如何累积

```c
// 状态预测：x(k+1) = Φ * x(k)
// 由于x初始为0，预测后仍接近0

// 协方差预测：P(k+1) = Φ * P(k) * Φ' + Q
// Q代表过程噪声（IMU噪声导致的不确定性累积）
lcUpdatePhiPmPhit(lcconfig, lcdata, tmp1X2, tmp0X2, lcdata->pmatpd);
matrixAdd(lcdata->rank, lcdata->rank, lcdata->pmatpd, tmp2X2, lcdata->pmatpd);
```

P矩阵不断增大，表示惯导误差在累积。

#### 7.1.3 更新阶段：误差如何被"发现"

当GNSS观测到来时，通过**新息**发现误差：

```c
// 新息 = INS预测位置 - GNSS观测位置
zgnss[0] = tmp3[0] - lcdata->gnssdata.pos[0];  // 纬度差（这就是误差！）
zgnss[1] = tmp3[1] - lcdata->gnssdata.pos[1];  // 经度差
zgnss[2] = lcdata->gnssdata.pos[2] - tmp3[2];  // 高度差
```

卡尔曼更新估计误差状态：

```c
// K = P * H' * (H * P * H' + R)^(-1)  ← P决定对INS和GNSS的信任程度
// x = x + K * z                       ← z是新息，K*z是最优误差估计
```

#### 7.1.4 反馈阶段：误差如何被修正

将估计的误差反馈到名义状态：

```c
static void lcKalmanFeedback(LcData *lcdata) {
    // 位置反馈
    lcdata->pvacur.pos[2] += lcdata->xmatpd[2];
    
    // 速度反馈
    // ...
    
    // 姿态反馈
    dtheta[0] = lcdata->xmatpd[6] + rv[0];
    // ...
    
    // IMU零偏反馈
    for (k = 0; k < 3; k++) {
        lcdata->imudata.gyrbias[k] += lcdata->xmatpd[9 + k];
        lcdata->imudata.accbias[k] += lcdata->xmatpd[12 + k];
    }
}
```

### 7.2 误差状态卡尔曼滤波流程图

```
┌─────────────────────────────────────────────────────────────────────┐
│                    误差状态卡尔曼滤波 (ESKF) 流程                     │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  IMU数据 ──┬──► 惯导机械编排 ──► INS名义状态 ─────────────┐          │
│            │      (位置/速度/姿态)      (带误差累积)      │          │
│            │                                              ▼          │
│            │                               ┌───────────────────┐    │
│            │                               │   计算新息         │    │
│            │                               │ z = INS位置 -     │    │
│            │                               │     GNSS位置      │    │
│  GNSS数据 ─┼──────────────────────────────►│  (直接观测误差!)  │    │
│            │                               └─────────┬─────────┘    │
│            │                                         │              │
│            │                                         ▼              │
│            │                               ┌───────────────────┐    │
│            │    ┌──────────────────────────│  卡尔曼更新        │    │
│            │    │      协方差P决定K        │ K = f(P, R)       │    │
│            │    │                          │ δx = K × z        │    │
│            │    │                          │ (估计误差状态)     │    │
│            │    │                          └─────────┬─────────┘    │
│            │    │                                    │              │
│            │    ▼                                    ▼              │
│     ┌──────────────────┐                  ┌───────────────────┐    │
│     │  协方差预测       │                  │  误差反馈         │    │
│     │ P = Φ*P*Φ' + Q   │◄─────────────────│ 名义状态 -= δx    │    │
│     │ (不确定性累积)    │     P更新后减小   │ 零偏 += δbg      │    │
│     └────────┬─────────┘                  │ δx = 0 (清零)     │    │
│              │                            └───────────────────┘    │
│              │                                                      │
│              └──────────────► 下一时刻预测                          │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

### 7.3 关键理解

1. **误差状态初始为0**：假设初始名义状态准确
2. **协方差P不为0**：表示对名义状态的不确定性
3. **P的作用是权衡**：决定对INS和观测各信任多少
4. **新息z是误差的直接观测**：告诉我们误差有多大
5. **K×z给出最优误差估计**：结合P和R的加权结果
6. **反馈后误差清零**：误差已补偿到名义状态

## 8. 误差状态清零机制深入解析

### 8.1 问题1：误差状态清零后，预测时状态是否为0？

#### 8.1.1 关键答案

**是的，每次更新后误差状态清零，下次预测时从0开始。** 但这不影响系统的正确性，因为：

1. 误差已经被反馈到**名义状态**中
2. 预测的是**新产生的误差**，而非累积误差
3. 协方差P记录了不确定性，会正常传递

#### 8.1.2 代码分析

在本项目中，更新流程如下：

```c
void lcUpdateFeedback(LcData *lcdata) {
    // 步骤1: 更新前将误差状态清零
    memset(lcdata->xmatud, 0, KALMANX * sizeof(double));  // x = 0
    
    // 步骤2: 执行卡尔曼更新，计算新的误差估计
    // x = x + K * (z - H*x) = 0 + K*z = K*z
    lcKalmanUpdate(lcdata, lcdata->updaterows);
    
    // 步骤3: 将误差反馈到名义状态
    // 名义状态 -= 误差估计
    lcKalmanFeedback(lcdata);
    
    // 反馈后，误差已补偿，x隐式变为0
    // （实际代码中没有显式清零，因为下次更新前会清零）
}
```

#### 8.1.3 时序示意图

```
时刻k:
  ├─ 预测: x(k) = Φ * x(k-1) = Φ * 0 = 0
  │        P(k) = Φ * P(k-1) * Φ' + Q   (P增大)
  │
  ├─ 无观测: 继续预测...
  │
时刻k+n (有GNSS观测):
  ├─ 更新前: x = 0, P 已累积
  │
  ├─ 计算新息: z = INS位置 - GNSS位置 = 5米
  │
  ├─ 卡尔曼更新: x = 0 + K*z = K*5 = 4米 (误差估计)
  │              P = (I-KH)*P           (P减小)
  │
  ├─ 反馈: INS位置 -= 4米              (修正名义状态)
  │        零偏 += δbg                  (累积零偏估计)
  │
  └─ 反馈后: x = 0 (概念上清零，误差已补偿)
  
时刻k+n+1:
  └─ 预测: x = Φ * 0 = 0  (从0开始预测新误差)
           P = Φ * P * Φ' + Q (继续累积不确定性)
```

#### 8.1.4 为什么这样做是正确的？

**关键理解**：误差状态估计的是**名义状态与真实状态的差**。

- 反馈前：名义状态有误差 → 误差状态 = δx
- 反馈后：名义状态 -= δx → 名义状态接近真实 → 误差状态应该 = 0

下一时刻预测的是**新产生的误差**（由IMU噪声等引起），而非历史误差。

### 8.2 问题2：为什么状态清零后协方差矩阵数值更稳定？

#### 8.2.1 直接估计状态的数值问题

如果直接估计位置、速度等状态，会遇到以下问题：

##### 问题1：数值量级差异巨大

| 状态 | 典型值 | 量级 |
|------|--------|------|
| 纬度 | 0.5236 rad (30°) | 10⁻¹ |
| 经度 | 2.0944 rad (120°) | 10⁰ |
| 高度 | 100 m | 10² |
| 东向速度 | 10 m/s | 10¹ |
| 陀螺零偏 | 1e-5 rad/s | 10⁻⁵ |

协方差矩阵P中元素的量级差异达到**10⁷**以上，矩阵求逆时容易产生数值误差。

##### 问题2：状态转移导致数值增长

```
位置状态：r(k+1) = r(k) + v*dt
```

状态值不断累加，数值越来越大，协方差矩阵的条件数恶化。

#### 8.2.2 误差状态的数值优势

使用误差状态后：

##### 优势1：误差始终在小范围内

| 误差状态 | 典型值 | 量级 |
|----------|--------|------|
| 位置误差 | 0.1 ~ 10 m | 10⁰ ~ 10¹ |
| 速度误差 | 0.01 ~ 0.5 m/s | 10⁻² ~ 10⁻¹ |
| 姿态误差 | 0.001 ~ 0.01 rad | 10⁻³ ~ 10⁻² |
| 零偏误差增量 | 1e-6 ~ 1e-4 rad/s | 10⁻⁶ ~ 10⁻⁴ |

反馈清零后，误差状态重置，量级始终可控。

##### 优势2：协方差矩阵条件数更好

```
直接状态:  P 的条件数可达 10¹⁴ 以上 → 矩阵求逆不稳定
误差状态:  P 的条件数通常在 10⁶ 以下 → 数值稳定
```

##### 优势3：避免累积舍入误差

```c
// 直接状态：每次状态转移都在大数上做加法
r(k+1) = r(k) + v*dt  // 大数 + 小数 → 舍入误差累积

// 误差状态：每次从0开始
δr(k+1) = Φ * 0 + 过程噪声  // 小数运算 → 舍入误差小
```

#### 8.2.3 数值稳定性对比示例

假设GPS周秒 sow = 604800.123（约一周），直接使用会导致：

```
直接状态：
  时间相关计算涉及 604800.123 级别的数值
  与 0.01 秒的增量相加时，有效精度损失

误差状态：
  只关心 Δt = 0.01 秒
  所有运算在小数范围内，精度有保证
```

#### 8.2.4 代码中的数值稳定措施

```c
// 协方差矩阵对称化，防止数值误差累积导致非对称
void matrixSymmetric(int n, double *mat) {
    for (int i = 0; i < n; i++) {
        for (int j = i+1; j < n; j++) {
            double avg = 0.5 * (mat[i*n+j] + mat[j*n+i]);
            mat[i*n+j] = avg;
            mat[j*n+i] = avg;
        }
    }
}
```

### 8.3 总结

| 问题 | 答案 |
|------|------|
| 清零后预测状态是0吗？ | **是的**，预测从0开始，因为历史误差已反馈到名义状态 |
| 为什么状态清零后P更稳定？ | 误差状态量级小且可控，避免大数运算，条件数好 |

### 8.4 误差状态 vs 直接状态 总对比

| 特性 | 直接状态估计 | 误差状态估计 |
|------|-------------|-------------|
| 状态量级 | 大（位置可达10⁶米） | 小（误差通常<10米） |
| 协方差条件数 | 高（10¹⁴+） | 低（<10⁶） |
| 矩阵求逆稳定性 | 差 | 好 |
| 舍入误差累积 | 严重 | 轻微 |

| 与INS配合 | 需额外处理 | 天然配合 |
